# Codex 项目规则

## 项目目标

这个仓库是一个以教学为导向的 LangChain Demo。

添加到这个项目中的所有功能，都应当优先服务于学习目标：

- 展示某一种 LangChain 能力
- 展示 LangChain 组件如何在真实工作流中组合使用
- 保持行为易于理解、易于观察、易于验证

## 功能规则

在添加新功能时：

- 优先考虑教学清晰度，而不是产品完整度
- 说明或保留该功能所要体现的 LangChain 概念
- 在有助于学习时，同时保留确定性实现和基于 LangChain 的实现，便于对照
- 避免引入那些不能提升 LangChain 学习价值的业务复杂度

## 实现偏好

- 优先采用小而清晰的模块划分，使其能够明确映射到 LangChain 概念
- 保持日志、示例和 CLI 流程对学习者友好、易于跟踪
- 仅在确实有助于展示 LangChain 用法时，才引入真实的外部集成
- 在 README 中将每个新增能力都作为学习示例来说明，而不只是作为功能罗列
